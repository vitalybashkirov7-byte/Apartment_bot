# Документация по Telegram-боту поиска квартир в Новосибирске

## Обзор

Telegram-бот автоматически ищет квартиры в Новосибирске на четырёх сайтах:
- **Циан** (cian.ru) — ✅ Playwright fallback (HTTP заблокирован)
- **Авито** (avito.ru) — ⚠️ undetected-chromedriver (IP может быть заблокирован)
- **N1.RU** (n1.ru) — ✅ Playwright fallback (HTTP заблокирован)
- **Домклик** (domclick.ru) — ⚠️ undetected-chromedriver (нестабильно)

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

Дополнительно для обхода блокировок:
```bash
pip install curl_cffi
```

### 2. Настройка окружения

Скопируйте `.env.example` в `.env` и заполните:

```env
TELEGRAM_BOT_TOKEN=ваш_токен
DEV_CHAT_ID=ваш_telegram_id
MIN_AREA=90
MAX_PRICE=17000000
MIN_ROOMS=3
MIN_BATHROOMS=2
RATE_LIMIT_PER_MINUTE=5
DAILY_LIMIT_PER_USER=50
LOG_LEVEL=INFO
```

### 3. Запуск

```bash
python main.py
```

## Модули

### config.py

Загрузка конфигурации:
- Приоритет: переменные окружения → зашифрованный файл → .env
- Параметры поиска, лимиты, прокси, логирование

### parser.py

Модуль парсинга:
- `Apartment` — dataclass модели квартиры
- `parse_cian()` — HTTP-парсер Циан (aiohttp + BeautifulSoup)
- `parse_cian_playwright()` — Playwright fallback для Циан (region=4897 = Новосибирск)
- `parse_avito_playwright()` — UC non-headless парсер Авито
- `parse_n1()` — HTTP-парсер N1.RU
- `parse_n1_playwright()` — Playwright fallback (url: novosibirsk.n1.ru/kupit/kvartiry/)
- `parse_domclick_playwright()` — UC non-headless парсер Домклик
- `fetch_page_with_retry()` — загрузка с повторными попытками
- `is_decent_apartment()` — фильтрация "приличных" квартир
- `_kill_chrome()` — очистка Chrome-процессов между UC-парсерами

### bot.py

Telegram-бот:
- `ApartmentBot` — основной класс
- `_do_search()` — поиск с прогрессом и логом ошибок
- `_test_parsers()` — тест всех парсеров (DEBUG-режим)
- `create_bot(proxy=...)` — создание бота с прокси для Telegram API
- Обработчики команд и callback-запросов
- Инлайн-клавиатуры (ВКЛЮЧАЯ "🧪 Тест парсеров" в DEBUG)
- Валидация ввода
- Таймаут 5 минут

### stats.py

Статистика:
- `track_request()` — трекинг запросов
- `track_user()` — трекинг пользователей
- `get_dashboard()` — форматирование дашборда
- Хранение в `bot_stats.json`

### user_settings.py

Хранение настроек:
- `get_user_settings()` — получить настройки (с дефолтами)
- `update_user_setting()` — обновить параметр
- Сохранение в `user_settings.json`

### main.py

Точка входа:
- Настройка логирования с ротацией (MAX_LOG_SIZE_MB)
- Проверка токена
- Запуск polling

## Решение проблем

### Блокировка сайтов

Все 4 сайта блокируют HTTP-парсинг. Используются две стратегии:

1. **Playwright fallback** (CIAN, N1.RU) — headless-браузер рендерит JavaScript.
   Если HTTP возвращает 0 результатов, автоматически запускается Playwright.

2. **undetected-chromedriver** (Авито, Домклик) — non-headless браузер обходит
   Qrator/anti-bot. UC-парсеры запускаются последовательно с очисткой Chrome
   между ними (`_kill_chrome()`).

### Прокси (Karing / VPN)

Прокси используется **только для Telegram API**, парсеры работают напрямую.

```
# Включить прокси (Karing запущен):
USE_PROXY=true

# Выключить прокси (парсеры работают, Telegram — нет):
USE_PROXY=false
```

**Как это работает:**
- `config.py` **всегда** очищает env-переменные прокси (HTTP_PROXY и т.д.)
- Прокси для Telegram передаётся **явно** через `HTTPXRequest(proxy=...)` в `create_bot()`
- Парсеры **не видят** прокси в env → не ломаются
- Karing можно включать/выключать — достаточно переключить `USE_PROXY`

**Порты Karing:**
- HTTP: 10809
- SOCKS5: 10808
- Альтернативные: 3065-3067

### Тестирование парсеров

В DEBUG-режиме (`LOG_LEVEL=DEBUG`) в меню бота появляется кнопка **"🧪 Тест парсеров"**:
- Последовательно запускает CIAN → N1.RU → Авито → Домклик
- Показывает прогресс в реальном времени
- Выводит количество квартир, время и примеры

### Задержки

Увеличьте `REQUEST_DELAY` если сайты блокируют частые запросы.

### Бот не отвечает

1. Проверьте токен: `python -c "from config import TELEGRAM_BOT_TOKEN; print(TELEGRAM_BOT_TOKEN)"`
2. Проверьте логи: `bot.log`
3. Убедитесь, что нет другого экземпляра бота
4. Проверьте конфликты в логах: `Conflict: terminated by other getUpdates request`

### Нет результатов поиска

1. Проверьте логи — бот показывает причину для каждого сайта
2. Возможные причины:
   - HTTP 429 — блокировка (подождите или добавьте прокси)
   - HTTP 401 — требуется авторизация (Домклик)
   - 0 объявлений — селекторы устарели
3. Нажмите «🔄 Обновить поиск» для повторной попытки

### Квартиры найдены, но не показаны

Бот объясняет причину:
```
⚠️ Найденные квартиры не прошли фильтр:
  • ул. Примерная, 5 — нет балкона, состояние: среднее
```

## Безопасность

- Не публикуйте токен бота и `.env` файл
- Используйте шифрование секретов (`secrets_crypto.py`)
- Не парсите сайты слишком frequently
- Соблюдайте лимиты запросов
